# Chat Room App

A Flutter chat application built with Firebase Authentication and Cloud Firestore.

The app lets users create an account, sign in with email and password, browse other registered users, and exchange direct messages in real time. It also includes light and dark themes managed with Provider.

## Features

- Email and password authentication with Firebase Auth
- Account registration and login screens
- Authentication gate that automatically routes users based on session state
- Firestore-backed user directory
- One-to-one chat rooms between registered users
- Real-time message updates ordered by timestamp
- Light and dark mode toggle
- Reusable UI components for buttons, text fields, user tiles, chat bubbles, and navigation drawer
- Firebase configuration for Android, iOS, macOS, web, and Windows

## App Flow

1. `main.dart` initializes Firebase and registers the app providers.
2. `AuthGate` listens to Firebase Auth state changes.
3. Signed-out users see the login/register flow.
4. Signed-in users land on the home page.
5. The home page displays all registered users except the current user.
6. Selecting a user opens a direct chat page.
7. Messages are written to and streamed from Cloud Firestore in real time.

## Firestore Data Model

The app uses two main Firestore areas:

### Users

Each authenticated user is stored in the `Users` collection.

```text
Users/{uid}
├── uid: string
└── email: string
```

### Chat Rooms

Direct chat rooms are stored in the `chat_rooms` collection. A room ID is generated by sorting both user IDs and joining them with an underscore, which ensures both users resolve to the same room.

```text
chat_rooms/{userA_userB}/messages/{messageId}
├── senderID: string
├── senderEmail: string
├── receiverID: string
├── message: string
└── timestamp: Timestamp
```

## Getting Started

### Prerequisites

- Flutter SDK compatible with Dart `^3.11.3`
- A Firebase project with Authentication and Cloud Firestore enabled
- FlutterFire CLI if you need to regenerate Firebase configuration files

### Installation

Clone the project and install the dependencies:

```bash
flutter pub get
```

### Firebase Setup

This repository already contains Firebase configuration files generated for the existing Firebase project:

- `lib/firebase_options.dart`
- `android/app/google-services.json`
- `ios/Runner/GoogleService-Info.plist`
- `macos/Runner/GoogleService-Info.plist`

If you want to connect the app to a different Firebase project, run:

```bash
flutterfire configure
```

Then enable these Firebase products in the Firebase Console:

- Authentication with the Email/Password provider
- Cloud Firestore

## Running the App

Run the app on a connected device, emulator, simulator, or supported desktop/web target:

```bash
flutter run
```

For Android release builds:

```bash
flutter build apk --release
```

## Testing and Analysis

Run static analysis:

```bash
flutter analyze
```

Run the Flutter test suite:

```bash
flutter test
```

## Main Files

- `lib/main.dart` initializes Firebase and registers providers.
- `lib/service/auth/auth_service.dart` handles sign in, sign up, sign out, and user persistence.
- `lib/service/auth/auth_gate.dart` routes users based on authentication state.
- `lib/service/chat/chat_service.dart` reads users, sends messages, and streams messages.
- `lib/pages/home_page.dart` displays available users.
- `lib/pages/chat_page.dart` displays and sends direct messages.
- `lib/pages/settings_page.dart` contains the dark mode switch.
- `lib/themes/theme_provider.dart` manages the active app theme.

## Notes

- The app currently uses email addresses as the visible user identity.
- Messages are only sent when the message input is not empty.
- The current user is hidden from the home page user list.
- The Android manifest includes the internet permission required for Firebase networking.
